# Travel Wallet App 🌍✈️

пет проект
Кроссплатформенное мобильное приложение на Flutter для отслеживания личных финансов и расходов в международных путешествиях. 

---

## 🚀 Основной функционал
* **Мониторинг курсов валют:** Отображение актуальных курсов мировых валют, необходимых в поездках (Юани, Донги, Доллары и др.) по отношению к базовой валюте.
* **Мультивалютные кошельки:** Просмотр детализации по конкретной стране/валюте (история трат, категории расходов).
* **Offline-режим:** Полная автономность в путешествиях благодаря локальному кэшированию данных.
* **Продвинутый мониторинг:** Сквозное логирование всех сетевых запросов и ошибок приложения.

---

## 🛠 Технический стек (Tech Stack)

Приложение построено с использованием промышленного стека библиотек, который применяется в крупных коммерческих проектах:

| Компонент | Технология / Пакет | Описание |
| :--- | :--- | :--- |
| **State Management** | `flutter_bloc` + `equatable` | Строгое управление состояниями на основе однонаправленного потока данных (UDF). |
| **Networking** | `dio` | Асинхронные HTTP-запросы к API, кастомные интерцепторы для обработки ответов. |
| **Local Storage** | `hive` | Высокопроизводительная NoSQL база данных для кэширования курсов и сохранения расходов в памяти устройства. |
| **Navigation** | `auto_route` | Декларативная навигация с использованием автоматической генерации маршрутов. |
| **Logger & Errors** | `talker` | Расширенное логирование сетевой активности (`DioLogHandler`), ошибок и состояний BLoC. |
| **Analytics** | `firebase_analytics` | Интеграция с сервисами Google для сбора метрик и аналитики поведения пользователей. |
| **Code Generation** | `json_serializable` \| `build_runner` | Автоматическая безопасная сериализация и десериализация входящих JSON-пакетов в Dart-модели. |

## 📂 Архитектурные подходы
* **Декомпозиция UI:** Интерфейс разбит на независимые, переиспользуемые виджеты (`Stateless`/`Stateful`).
* **Разделение слоев:** Бизнес-логика (`BLoC`) полностью изолирована от слоя представления (`View`).
* **Абстракция данных:** Работа с сетью и хранилищем скрыта за интерфейсами репозиториев (инъекция зависимостей), что делает код тестируемым и гибким к изменениям.
* **Безопасная типизация:** Отказ от работы с сырыми `Map` структурами в пользу строго типизированных моделей данных.
